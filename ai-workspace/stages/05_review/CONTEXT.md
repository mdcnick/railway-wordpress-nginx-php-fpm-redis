# 05_review stage

## Purpose

Check the work before delivery.

## Inputs

- `output/plan.md`
- `output/build-summary.md`
- changed files
- `package.json` scripts or project commands

## Process

- Run safe relevant checks if available.
- Review changed files.
- Compare implementation against plan.
- Note anything not completed.
- Do not hide failures.

## Outputs

- `output/review.md`

## Allowed tools

- File read/search tools for the listed inputs.
- Markdown editing tools for stage outputs and workspace docs.
- Safe project commands only when the stage process calls for them.
- No destructive commands without explicit permission.


## Skills to use

- `testing-review.md`

## Done criteria

- The result is checked, and remaining risks are documented.
