# 01_learn stage

## Purpose

Learn the repository structure and document it.

## Inputs

- Repo root
- `package.json` or equivalent config files
- existing README/docs
- `permissions.md`

## Process

- Inspect the repo without changing code.
- Identify where the actual code lives.
- Identify commands, stack, and important folders.
- Update repo-map and learned context.

## Outputs

- Updated `_config/repo-map.md`
- Updated `_config/learned-context.md`
- `output/repo-learning-summary.md`

## Allowed tools

- File read/search tools for the listed inputs.
- Markdown editing tools for stage outputs and workspace docs.
- Safe project commands only when the stage process calls for them.
- No destructive commands without explicit permission.


## Skills to use

- `repo-analysis.md`

## Done criteria

- A future agent can understand the repo without scanning everything again.
