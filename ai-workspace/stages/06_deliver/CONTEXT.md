# 06_deliver stage

## Purpose

Summarize final work for the user.

## Inputs

- `output/intake.md`
- `output/plan.md`
- `output/build-summary.md`
- `output/review.md`

## Process

- Give a concise summary.
- List changed files.
- List checks run.
- List known issues or next steps.
- Save final delivery note.

## Outputs

- `output/final-summary.md`
- `outputs/latest-delivery-summary.md`

## Allowed tools

- File read/search tools for the listed inputs.
- Markdown editing tools for stage outputs and workspace docs.
- Safe project commands only when the stage process calls for them.
- No destructive commands without explicit permission.


## Skills to use

- `documentation-writer.md`

## Done criteria

- A human can understand what happened and what to do next.
