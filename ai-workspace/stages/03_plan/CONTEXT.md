# 03_plan stage

## Purpose

Create a focused plan before editing.

## Inputs

- `output/intake.md`
- `_config/repo-map.md`
- `_config/coding-standards.md`
- relevant source files if needed

## Process

- Identify files likely to change.
- Create a small plan.
- Identify risks.
- Avoid broad rewrites.
- Define how the work will be checked.

## Outputs

- `output/plan.md`

## Allowed tools

- File read/search tools for the listed inputs.
- Markdown editing tools for stage outputs and workspace docs.
- Safe project commands only when the stage process calls for them.
- No destructive commands without explicit permission.


## Skills to use

- `feature-build.md`
- `bug-debugging.md`
- `code-refactor.md`

## Done criteria

- The build stage has a clear plan.
