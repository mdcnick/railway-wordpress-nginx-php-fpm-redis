# 00_leadership stage

## Purpose

Define how leadership roles guide the agent's work.

## Inputs

- `_config/project.md`
- `_config/leadership.md`
- User answers

## Process

- Ask only useful questions.
- Clarify CEO, COO, CMO, and optional CTO/Product roles.
- Save durable leadership preferences.

## Outputs

- Updated `_config/leadership.md`
- `output/leadership-notes.md`

## Allowed tools

- File read/search tools for the listed inputs.
- Markdown editing tools for stage outputs and workspace docs.
- Safe project commands only when the stage process calls for them.
- No destructive commands without explicit permission.


## Skills to use

- `documentation-writer.md`

## Done criteria

- Leadership roles are clear enough to guide future decisions.
