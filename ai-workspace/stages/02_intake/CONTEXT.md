# 02_intake stage

## Purpose

Clarify the task before planning or building.

## Inputs

- User request
- `_config/project.md`
- `_config/repo-map.md`
- `_config/learned-context.md`

## Process

- Restate the task.
- Identify task type: bug, feature, refactor, docs, research, strategy, content, or cleanup.
- Identify likely files or folders involved.
- Ask questions only if blocked.
- Otherwise proceed with assumptions and document them.

## Outputs

- `output/intake.md`

## Allowed tools

- File read/search tools for the listed inputs.
- Markdown editing tools for stage outputs and workspace docs.
- Safe project commands only when the stage process calls for them.
- No destructive commands without explicit permission.


## Skills to use

- `documentation-writer.md`

## Done criteria

- The task is clear enough to plan.
