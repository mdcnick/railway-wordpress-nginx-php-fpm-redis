# Testing review skill

Use this when checking work before delivery.

## Process

1. Identify available commands.
2. Run relevant safe checks.
3. Inspect changed files.
4. Compare the result against the plan.
5. Report failures honestly.
6. Suggest next fixes when needed.

## Rules

- Do not suppress tests to pass.
- Do not claim checks passed unless they were run.
- Prefer targeted checks for the files or behavior changed.
