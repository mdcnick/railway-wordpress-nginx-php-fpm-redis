# Code refactor skill

Use this for cleanup, simplification, or structure changes.

## Process

1. State the behavior that must be preserved.
2. Identify duplication, awkward boundaries, or confusing code.
3. Keep changes small.
4. Avoid premature abstraction.
5. Check behavior after changes.
6. Document any durable pattern learned.

## Rules

- Preserve behavior.
- Do not mix refactor with feature changes unless asked.
- Do not rename public interfaces without checking call sites.
