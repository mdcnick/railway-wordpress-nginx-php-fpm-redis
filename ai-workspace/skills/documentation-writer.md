# Documentation writer skill

Use this for markdown, summaries, plans, and durable workspace notes.

## Process

1. Write clear markdown.
2. Keep docs useful and short.
3. Prefer examples over abstract explanations.
4. Use headings that help scanning.
5. Update durable context when useful.
6. Avoid repeating details that belong in a source instruction file.

## Rules

- Plain language.
- No filler.
- No invented business direction.
