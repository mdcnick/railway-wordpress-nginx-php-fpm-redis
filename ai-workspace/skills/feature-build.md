# Feature build skill

Use this for new behavior or meaningful product changes.

## Process

1. Understand desired behavior.
2. Identify the data flow.
3. Find existing patterns.
4. Implement the minimal complete version.
5. Add or update tests if the repo has test patterns.
6. Check the result with relevant commands or scenarios.
7. Document important decisions.

## Rules

- Do not ship placeholders.
- Do not create new architecture unless the feature needs it.
- Keep unrelated files unchanged.
