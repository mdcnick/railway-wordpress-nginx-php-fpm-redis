2. **railway.js lines 82-84** — Add `healthcheckPath: '/health'` to the input object alongside `rootDirectory: '/'`. Do NOT create a second serviceInstanceUpdate call.

3. **default.conf.template lines 60-65** — Replace the entire `/health` location block with:
```nginx
    # Health check endpoint — proves both Nginx and PHP-FPM are alive
    location = /health {
        access_log off;
        fastcgi_pass 127.0.0.1:9000;
        fastcgi_param SCRIPT_FILENAME $document_root/health.php;
        fastcgi_param QUERY_STRING "";
        include fastcgi_params;
    }
```
Note: use exact match `=` (not prefix match). This replaces the old static `return 200` response.

4. **docker-entrypoint.sh** — Add a new step between step 5 (wp-config injection, ends around line 50) and step 6 (fix permissions, line 52-53). Insert:
```bash
# 5.5. Write health check script (must be after WP init so volume exists)
echo "Writing health check script..."
echo '<?php header("Content-Type: application/json"); echo json_encode(["status" => "ok"]);' > /var/www/html/health.php
```
This ensures health.php is present on every container start regardless of volume state. Do NOT use Dockerfile COPY for this file — the volume mount would shadow it.
