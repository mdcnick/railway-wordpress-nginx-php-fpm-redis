## Directory Purposes

**.github:**
- Purpose: GitHub and CI/CD configuration
- Contains: Workflow definitions, dependency management rules
- Key files: `dependabot.yml` - automated dependency update configuration

**.planning:**
- Purpose: Project planning and codebase analysis
- Contains: Analysis documents for architecture, structure, conventions, testing, tech stack, and concerns
- Generated by: GSD codebase mapping commands

**Root (Project Root):**
- Purpose: Docker container configuration and deployment templates
- Contains: Docker build instructions, entrypoint scripts, Nginx configuration, WordPress customizations
- Key files: `Dockerfile`, `docker-entrypoint.sh`, configuration templates

## Key File Locations

**Entry Points:**

- `Dockerfile`: Container image build definition - specifies base image (wordpress:6-php8.3-fpm-alpine), installs dependencies, configures services, copies configuration files
- `docker-entrypoint.sh`: Container initialization script - generates configs, injects customizations, starts Nginx and PHP-FPM processes
- `nginx.conf`: Nginx main configuration - sets worker processes, includes server block config
- `default.conf.template`: Nginx server configuration - handles all routing, security rules, FastCGI proxying to PHP-FPM

**Configuration:**

- `.env.example`: Template showing required and optional environment variables for Railway services
- `wp-config-custom.php`: Custom WordPress configuration - dynamic domain detection and Redis connection setup
- `docker-entrypoint.sh`: Dynamic configuration generation - creates PHP and Nginx configs from environment variables

**Core Logic:**

- `/var/www/html/wp-config.php`: Generated WordPress configuration (in running container, not in repo) - includes injected custom config
- `/var/www/html/`: Standard WordPress installation directory (mounted volume, not in repo)

**Application Layer:**

- `nginx.conf` + `default.conf.template`: Web server and routing logic
- `wp-config-custom.php`: Custom application-level configuration
- `/usr/local/share/wp-config-custom.php`: Injected custom config file location (in container)

## Naming Conventions

**Files:**

- **Docker configuration:** `Dockerfile` (standard naming)
- **Shell scripts:** `docker-entrypoint.sh` (kebab-case for multi-word names)
- **Nginx configuration:** `nginx.conf` (lowercase), `default.conf.template` (template suffix)
- **PHP configuration:** `wp-config-custom.php` (WordPress standard `wp-` prefix, custom suffix)
- **Documentation:** `README.md`, `RAILWAY_TEMPLATE.md` (UPPERCASE.md for primary docs)
- **Environment:** `.env.example`, `.env` (dotfile prefix, lowercase)
- **GitHub:** `.github/dependabot.yml` (dotfile directory, hyphenated filenames)

**Directories:**

- **Hidden directories:** `.github/`, `.planning/` (dotfile prefix for configuration/meta)
- **Lowercase:** No subdirectories in current structure beyond single-level organization
- **Functional grouping:** `.github/` for version control tools, `.planning/` for project planning

## Where to Add New Code

**WordPress Plugins/Themes:**
- Location: Container volume mount at `/var/www/html/wp-content/plugins/` or `/var/www/html/wp-content/themes/`
- Method: Upload via WordPress admin panel or use WP-CLI
- Persistence: Mounted volume ensures persistence across container restarts

**Custom PHP Code:**
- Location: Must-use plugins at `/var/www/html/wp-content/mu-plugins/` (auto-loaded)
- Or: Custom theme at `/var/www/html/wp-content/themes/custom-theme/`
- Method: Upload files via WordPress or SFTP if available
- Note: PHP execution blocked in uploads directory per security policy (line 49-51 in `default.conf.template`)

**Configuration Additions:**
- Location: If modifying startup config, edit `wp-config-custom.php` and rebuild Docker image
- Location: If adding new environment variables, update `.env.example` and `docker-entrypoint.sh`
- Method: Commit to repo, rebuild image, redeploy container

**Nginx Security Rules:**
- Location: `default.conf.template` in server block
- Method: Add new `location` directive before serving content
- Example: To block a path, add:
  ```nginx
  location = /forbidden {
      deny all;
  }
  ```

**Nginx Performance Rules:**
- Location: `default.conf.template` - caching, compression, headers
- Method: Modify gzip settings (lines 14-18) or add caching rules (lines 53-58)

**Container Startup Logic:**
- Location: `docker-entrypoint.sh`
- Method: Add new bash commands following existing pattern (numbered sections)
- Note: Must handle errors with `set -e` and proper cleanup in trap handler

**Dependencies/Extensions:**
- Location: `Dockerfile` - add to apk install section or pecl/docker-php-ext-install commands
- Method: Modify RUN instructions and rebuild image
- Example: To add ImageMagick, add `imagemagick` to apk install (line 4-15)

## Special Directories

**Container Runtime Volumes:**
- `/var/www/html` - WordPress application files (mounted persistent volume)
- `/var/cache/nginx` - Nginx cache directory (created with proper permissions)
- `/var/lib/nginx` - Nginx library files (created with proper permissions)
- `/var/log/nginx` - Nginx logs (created with proper permissions)
- `/run/nginx` - Nginx runtime files (created with proper permissions)

**Container Mount Points:**
- Generated: No - all directories created during build
- Committed: Yes - `/var/www/html` is a persistent volume mounting point, not a directory with committed files

**Docker Image Base:**
- Source: `wordpress:6-php8.3-fpm-alpine` (official WordPress image from Docker Hub)
- Modifications: Custom Dockerfile layers add Nginx, dependencies, WP-CLI, and custom configuration
- Layer order: Base image → dependencies → extensions → custom config → entrypoint

## Security Boundary Locations

**Protected Paths (Nginx level):**
- `/xmlrpc.php` - Blocked entirely (lines 36-41 in `default.conf.template`)
- `/wp-config.php` - Blocked entirely (lines 43-46)
- `*/wp-content/uploads/*.php` - Blocked entirely (lines 48-51)
- `*/\.*` - Hidden files/directories blocked (lines 67-72)

**Allowed Static Assets (Cached):**
- Extensions: `.js`, `.css`, `.png`, `.jpg`, `.jpeg`, `.gif`, `.ico`, `.svg`, `.woff`, `.woff2`, `.ttf`, `.eot`
- Cache duration: 30 days browser cache
- Logged: Access log disabled for static assets

**Health Check Endpoint:**
- Path: `/health`
- Response: 200 OK with "healthy\n"
- Logging: Access log disabled
- Purpose: Container orchestration liveness/readiness probes
